# Translate

## Overview

Substitutes the translation corresponding to the given key using the currently selected
client-language while allowing to set placeholders in retrieved values as well as to provide
a fallback if there was no value with the given key; this tag acts as a placeholder and
must thereby be self-closing (does not accept content).

<TagDocRender tag={{
  name: 'translate',
  aliases: [],
  selfClosing: true,
  attributes: [
    {
      name: 'key',
      aliases: [],
      description: 'Path of the value to render',
      fallback: null,
      multiValue: false,
      type: AttributeType.STRING,
    },
    {
      name: 'fallback',
      aliases: [],
      description: 'Fallback value to use if key is absent',
      fallback: 'null',
      multiValue: false,
      type: AttributeType.STRING,
    },
    {
      name: 'with',
      aliases: [],
      description: 'Components rendered into placeholders',
      fallback: 'null',
      multiValue: true,
      type: AttributeType.MARKUP,
    },
  ]
}} />

## Examples

### Simple Value

Most values do not contain any placeholders, making for the simplest kind of translation.

```!component-markup
Buy <&b>32x <translate key="block.minecraft.blue_glazed_terracotta" /></> for <&a>15$
```

### Requiring Placeholders

Whenever placeholders are contained, specify them in the desired order; if values are missing, the client will not render the message.

```!component-markup
<translate
  key="death.attack.cactus.player"
  with={<&d>Notch}
  with={<&c>Herobrine}
/>
```
